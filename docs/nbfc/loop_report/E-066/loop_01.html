E-066 — Auto Anomaly Flag on NBFC record

Loop 01 — pass (3/3 ACs) — branch nbfc/E-066

Summary

Auto-evaluator that classifies every active NBFC tenant as red / amber / none based on per-tenant rollups derived from nbfc_loans (delinquency_pct, recovery_rate_pct, avg_dpd) and persists open flags in a new nbfc_anomaly_flags table. Run via the admin POST endpoint or the nightly Vercel cron at 00:10 UTC (after the compute-cds job seeds metric inputs). When a tenant's metrics return below thresholds, the open flag row is closed by stamping cleared_at = now(). New flags emit an audit-grade console.info for the iTarang Ops notification surface.

Thresholds (set autonomously per --auto-approve-schema)

  • delinquency_pct > 15 → breach
  • recovery_rate_pct < 70 → breach
  • avg_dpd > 30 → breach
  • 2 or 3 breaches ⇒ severity red
  • Exactly 1 breach ⇒ severity amber
  • 0 breaches ⇒ no flag (and clear any open flag)
