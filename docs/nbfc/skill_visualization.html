Human checkpoint

Four human checkpoints

  1. needs_clarification — BRD ambiguity at extract time. Human edits the YAML.
  2. needs_schema_approval — unit proposes new tables/columns. Human reviews per-unit checkpoint HTML, approves with --auto-approve-schema.
  3. needs_spec_review — spec-revalidate detects blocking BRD drift mid-iteration. Human approves the proposed YAML diff.
  4. needs_pr_review — green unit is ready to merge. Human confirms before git push.

Convergence guardrails

  • Inner loop: max 5 iterations / unit; halts on repeat-signature
  • Outer convergence: max 5 iterations; halts on stall
  • Spec amendments: max 2 per unit (3rd → blocked: spec_chase)
  • YAML revisions: max 3 per unit (4th → blocked: revision_chase)
  • Schema lock: at most 1 schema-modifying unit at a time

Headed scope (NEW)

Every passed unit is supposed to produce visual evidence:

  • UI ACs — page.goto() + page.screenshot() per AC step
  • API ACs — render JSON via /headed-api-viewer/[E-NNN]/page.tsx + screenshot
